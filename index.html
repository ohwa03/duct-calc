Filter Field Notes
• Typical 3-stage on critical AHU: MERV-7 pre → MERV-14 intermediate → HEPA final.
• Pre-filters protect HEPA service life — change on schedule.
• Any filter bypass gap eliminates protection regardless of HEPA condition — check gaskets/seals.
• OR: verify pressure differential AFTER filter change before returning room to service.
• Track filter ΔP — rising ΔP = loading = airflow drop. That's your PM trigger.

Containment Classes

Patient Risk (Low→Highest) × Project Type (A→D) = Class (I–V)

Construction/reno = Class III–V only. Higher class includes all lower-class controls.
Class I — Inspection / non-invasive
▾
Minimal dust. Replace displaced ceiling tile immediately.
Class II — Small / standing practice
▾
Wall patch, access panels. Dust control, mist for cutting, seal doors, wipe surfaces. Not for construction.
Class III — Permit ▲
▾
Sealed barriers + negative pressure HEPA. Isolate HVAC. Shut down VAV serving work area before covering returns/supplies. Sealed waste containers. Hold until IC + PM authorize removal.
Class IV — Permit ▲▲
▾
Class III + anteroom, HEPA-vac personnel before exit, patients removed, high-risk stay out 1 hr. Isolate return AND supply diffusers.
Class V — Highest risk ▲▲▲
▾
Procedural/invasive/sterile. Always containment barrier (NFPA 241). Full negative-pressure containment, anteroom, continuous monitoring.
Containment Pressure

Alarm setpoint: −0.01 to −0.05" w.g. (−2.5 to −12 Pa)

Negative flow: entry → anteroom → work zone. Monitor/document daily. HEPA runs until finishes done, HVAC operational, PM authorizes removal.

Reliable Controls · Control-BASIC crash course. Syntax varies slightly by controller/firmware — always check the actual program.
The Basics
Control BASIC runs in programs attached to a controller. Each program is lines of logic the controller scans top-to-bottom on a loop. You reference points (inputs, outputs, variables) and write conditional logic to control equipment.
Point Types
TypeMeaning
AIAnalog Input (sensor: temp, pressure)
AOAnalog Output (damper, valve 0–100%)
BIBinary Input (switch, status)
BOBinary Output (on/off relay)
AVAnalog Variable (setpoint, calc value)
BVBinary Variable (flag, true/false)
Core Syntax
IF / THEN logic:
IF AI1 > 75 THEN BO1 = ON IF AI1 < 70 THEN BO1 = OFF
Setpoint + deadband (avoid short cycling):
REM cooling call w/ 2 deg deadband IF SpaceTemp > (Setpoint + 1) THEN Cool = ON IF SpaceTemp < (Setpoint - 1) THEN Cool = OFF
Comments use REM:
REM this line is a comment
Program Structure
• Every line starts with a line number (10, 20, 30…). Leave gaps to insert later.
• Controller scans top→bottom every cycle, then repeats.
• KEYWORDS are UPPERCASE by convention.
• Colon : puts multiple statements on one line.
• Underscores in point names: AH01_FAN_CMD.
• RC-Studio won't download a program with syntax errors. Press F1 on any statement for its help.
REM colon = multiple statements per line 20 IF SCHED1 THEN START FAN ELSE STOP FAN 30 OPEN DAMPER : START LOOP1
Conditional Statements
StatementDoes
IF…THENRun action if condition true
IF…THEN…ELSERun one action or the other
IF+ …THENTrigger on RISING edge (false→true)
IF− …THENTrigger on FALLING edge (true→false)
GOTO nJump to line number n
GOSUB nJump to subroutine, RETURN comes back
RETURNReturn from a GOSUB
FOR…NEXTLoop a fixed number of times
ENDStop running the program this scan
IF+ / IF− are the edge-trigger operators — fire once on the transition, not every scan. Use them to reset a PID integral or pulse an event. Example: IF+ MODE_OCC THEN START LOOP1 resets the loop when the space goes occupied.

Control Loops (PID)
Loops are defined in their own section; your program just references them.
CmdDoes
START LOOPnRun loop / reset integral windup
STOP LOOPnFreeze the loop
LOOPn = spSet the loop's setpoint
CONPROPSet proportional band dynamically
CONRESETSet integral reset dynamically
CONRATESet derivative rate dynamically
Tip: don't START a loop every scan — it keeps resetting the integral and it'll never build proper corrective action. START it once on the occupied transition with IF+.
Trend / Alarm

• Alarm priorities can pop up, email, or link to a graphic.

VAV Box Example
REM dual-duct VAV — pick deck by demand IF SpaceTemp > ClgSP THEN ClgFlow = PID(...) IF SpaceTemp < HtgSP THEN HtgFlow = PID(...) REM K-factor converts sensor dP to flow ActualFlow = Kfactor * SQR(VelPressure)
That K × √VP is the same correction-factor math from the Duct tab — it's literally what the controller runs.
Field Tips
• Never edit a live program without backing up the original first.
• Watch for deadband — too tight = short cycling.
• Manual/override commands expire — confirm point returns to AUTO.
• On PM shutdowns, check the program isn't holding a value that breaks room pressure.
• Comment every change with REM + your initials + date.
